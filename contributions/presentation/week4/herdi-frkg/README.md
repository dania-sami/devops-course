## Description
In DevOps, a part of keeping a system running is collecting and analyzing metrics and logs to predict and proactively prevent any issues.
This can be done statically using pre-defined thresholds.
However, due to low dynamicity, we may get false positives.
A way to remedy this is by using machine learning models to dynamically predict future issues based on historical data and other constraints.
In our presentation, we therefore want to explain how AIOps works through prediction-based anomaly detection.

Our presentation will have the following format:
1. Introduce static metric and log based alerting
2. Highlight potential problems with this
3. Introduce AIOps and prediction-based anomaly detection, explaining what it is
4. Provide an example tool, [DataDog](https://www.datadoghq.com/blog/early-anomaly-detection-datadog-aiops/#scale-anomaly-detection-with-datadogs-machine-learning-algorithms)
5. Connect back to intro to showcase improvements
6. Reflect about positives and negatives

To showcase improvements, specifically related to using DataDog, we will use the following [blog](https://www.thoughtworks.com/insights/blog/generative-ai/from-alert-fatigue-to-AIOps-building-proactive-observability-stack-with-Datadog-on-EKS).

**Relevance**

A fundamental part of DevOps is keeping a service running.
Using predictive-based anomaly detection, we may find potential threats to this proactively and prevent it from happening.
Using AIOps for this, we may decrease false-positives through more refined and complex analysis.
