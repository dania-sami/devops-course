## Description

We plan to make a presentation on the Model Context Protocol (MCP). The contents of the presentation could be the following:

- Introduction: LLMs are great new tools that can be used for a variety of tasks. However, a problem is that LLMs cannot inherently communicate and interact with the tools we use. That is where MCP comes in. 
- Context: What MCP is, when was it created and by whom.
- Motivation: How MCP can be related to AIOps, i.e. using AI (in the form of an LLM here) to connect to our DevOps tools in order to effectivize the DevOps process. A specific and pedagogical example could be integrating a local LLM to fetch and compare new issues in GitHub to look for similarities and flag duplicates using MCP in the form of a MCP server.
- Reflection: Difficulties of doing the same thing if MCP wasn't a thing, also some general pros and cons. Maybe briefly highlight alternatives to MCP.

**Relevance**

We want to be able to apply state-of-the-art language models in our DevOps workflows, but the workflow and the LLM need a way to communicate, and that is where MCP comes in!
