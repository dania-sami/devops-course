## Description

In this presentation, we will explore the emerging field of LLMOps and how it differs significantly from traditional MLOps practices. While MLOps focuses on automating and managing the lifecycle of custom-trained machine learning models, LLMOps introduces new challenges when working with pre-trained, large-scale foundation models such as GPT, LLaMA, or Falcon.
 
We begin by defining both MLOps and LLMOps, followed by a comparison of their similarities and differences. We then deep dive into a key operational challenge specific to LLMs — evaluation and monitoring, where traditional ML metrics fall short. We highlight this challenge as critical for deploying LLM applications reliably in production.
 
To address this, we present LangSmith, an LLM observability and evaluation tool from the LangChain ecosystem. We demonstrate how LangSmith enables logging, evaluation, and human/LLM-in-the-loop feedback by instrumenting LLM pipelines. Our presentation includes a live code snippet walkthrough to show how prompt chains can be monitored and evaluated with custom criteria like “correctness” or “relevance.”
 
We conclude by reflecting on how LLMOps demands a rethinking of infrastructure, ethics, and lifecycle practices—moving from model-centric to experience-centric AI operations.

**Relevance**

This topic is highly relevant to DevOps because it expands DevOps and MLOps principles into the new frontier of LLM applications, which are increasingly being adopted in industry. LLMOps introduces new operational layers such as prompt management, observability for non-deterministic outputs, and feedback-driven evaluation loops. Understanding these concepts equips DevOps practitioners with the tools and mindset needed to support reliable, ethical, and scalable LLM-powered systems in production.
